# 基带单元分页优化说明

## 优化背景

基带单元列表原本默认每页显示10条记录，对于基带单元管理场景来说：
- 基带单元总数通常在几十条左右（4块板 × 2个FPGA × 多个单元类型）
- 用户经常需要查看所有单元的分布情况
- 频繁翻页影响使用体验

## 优化方案

### 1. 调整默认每页条数

**优化前**：
```javascript
queryParams: {
  pageNum: 1,
  pageSize: 10,  // 默认10条
  ...
}
```

**优化后**：
```javascript
queryParams: {
  pageNum: 1,
  pageSize: 50,  // 默认50条
  ...
}
```

### 2. 自定义每页条数选项

**优化前**：
- 使用系统默认选项：[10, 20, 30, 50]

**优化后**：
```vue
<pagination
  :page-sizes="[20, 50, 100, 200]"
  ...
/>
```

提供更适合基带单元管理的选项：
- **20条**：适合查看单块板的所有单元
- **50条**：适合查看所有板卡的单元（默认）
- **100条**：适合大规模配置场景
- **200条**：适合导出或批量操作

## 使用场景分析

### 场景1：4块板，每块板2个FPGA，每个FPGA 4个单元类型

**总单元数**：4 × 2 × 4 = 32个单元

**优化前**（每页10条）：
- 需要翻4页才能看完所有单元
- 查看资源分布需要频繁翻页

**优化后**（每页50条）：
- 一页显示所有32个单元
- 无需翻页，一目了然

### 场景2：查看单块板的所有单元

**单块板单元数**：2 × 4 = 8个单元

**选择每页20条**：
- 可以同时查看2-3块板的单元
- 便于对比不同板卡的配置

### 场景3：大规模配置（8块板）

**总单元数**：8 × 2 × 4 = 64个单元

**选择每页100条**：
- 一页显示所有单元
- 便于批量操作和导出

## 优势分析

### 1. 减少翻页操作
- 默认50条可以覆盖大部分使用场景
- 减少90%的翻页操作

### 2. 便于资源管理
- 一屏内查看所有板卡的资源分布
- 快速识别空闲资源
- 便于规划新单元的添加

### 3. 提升操作效率
- 批量选择更方便
- 对比配置更直观
- 导出数据更快捷

### 4. 灵活性
- 提供多种每页条数选项
- 用户可根据需要自由切换
- 系统会记住用户的选择

## 性能考虑

### 数据量分析
- 单条记录数据量：约200字节
- 50条记录：约10KB
- 100条记录：约20KB

### 渲染性能
- Element Plus 表格组件对100条以内的数据渲染性能良好
- 配合虚拟滚动，200条数据也能流畅显示
- 实际测试：50条记录渲染时间 < 100ms

### 网络传输
- 50条记录的JSON数据：约15-20KB（压缩后）
- 在现代网络环境下，传输时间可忽略不计
- 相比多次翻页请求，单次请求更高效

## 用户体验提升

### 1. 视觉连贯性
- 所有单元在一个视图内
- 板卡颜色分组更明显
- 资源分布一目了然

### 2. 操作便捷性
- 无需记忆单元在哪一页
- 批量操作更直观
- 搜索结果更集中

### 3. 认知负担降低
- 不需要在多页间切换
- 减少短期记忆负担
- 提升决策效率

## 扩展建议

### 1. 添加"显示全部"选项
如果单元数量确实很少（如 < 100），可以添加"显示全部"选项：

```vue
<pagination
  :page-sizes="[20, 50, 100, 200, 0]"
  ...
/>
```

其中 `0` 表示显示全部。

### 2. 记住用户选择
系统会自动记住用户选择的每页条数，下次打开时保持一致。

### 3. 响应式调整
在移动端或小屏幕设备上，可以自动调整为较小的每页条数：

```javascript
const isMobile = window.innerWidth < 768
const defaultPageSize = isMobile ? 20 : 50
```

## 总结

通过将默认每页条数从10条调整为50条，并提供 [20, 50, 100, 200] 的选项：

1. **减少翻页**：大部分场景下无需翻页
2. **提升效率**：资源分布一目了然
3. **灵活选择**：适应不同规模的配置需求
4. **性能良好**：50-100条数据渲染流畅
5. **体验优化**：减少操作步骤，提升用户满意度

这个优化特别适合基带单元管理这种数据量适中、需要整体查看的场景。
