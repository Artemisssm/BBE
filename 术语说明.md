# 基带监控系统术语说明

## 核心术语

### 1. 链路模式 (Link Mode)
**定义**：基带处理的工作模式，定义了数据传输的方向、速率和特性。

**8种链路模式**：
1. **返向中低速数传** - 返回链路，中低速数据传输
2. **返向高速数传** - 返回链路，高速数据传输
3. **ACM数传** - 自适应编码调制数据传输
4. **前向数传小环** - 前向链路，小环回测试
5. **低速模拟源** - 低速模拟信号源
6. **高速模拟源** - 高速模拟信号源
7. **ACM模拟源** - ACM模式模拟信号源
8. **前向数传** - 前向链路数据传输

**特点**：
- 每种链路模式包含4个单元类型（编码、调制、解调、译码）
- 不同链路模式的参数配置和约束规则不同
- 链路模式决定了整体的工作流程

### 2. 单元类型 (Unit Type)
**定义**：基带处理链路中的功能模块类型。

**4种单元类型**：
1. **编码单元 (ENCODE)** - 信道编码
2. **调制单元 (MODULATE)** - 信号调制
3. **解调单元 (DEMODULATE)** - 信号解调
4. **译码单元 (DECODE)** - 信道译码

**特点**：
- 每个单元类型有特定的参数定义
- 单元类型之间可以有约束关系
- 同一单元类型内部参数也可以有约束

### 3. 参数 (Parameter)
**定义**：单元类型的可配置属性。

**参数属性**：
- **参数代码** (paramCode) - 英文标识，如：ENC_CODE_TYPE
- **参数名称** (paramName) - 中文名称，如：编码方式
- **值类型** (valueType) - ENUM/UINT/FLOAT/SWITCH
- **取值范围** - 最小值、最大值
- **默认值** - 初始配置值

### 4. 参数约束 (Parameter Constraint)
**定义**：参数之间的依赖和限制关系。

**约束维度**：
- **链路模式** - 约束发生在哪个链路模式
- **单元类型** - 约束涉及哪些单元类型
- **参数** - 具体的源参数和目标参数

**约束类型**：
1. **值范围约束** - 限制参数取值范围
2. **启用/禁用约束** - 根据条件禁用参数
3. **值同步约束** - 参数值保持一致
4. **值计算约束** - 根据公式计算参数值

## 术语层次关系

```
链路模式 (8种)
  ├─ 编码单元
  │   ├─ 编码方式
  │   ├─ 编码码率
  │   └─ ...
  ├─ 调制单元
  │   ├─ 调制方式
  │   ├─ 符号率
  │   └─ ...
  ├─ 解调单元
  │   └─ ...
  └─ 译码单元
      └─ ...
```

## 约束关系示例

### 示例1：同链路、同单元类型
```
链路模式: 返向中低速数传
源单元类型: 调制单元
源参数: 调制方式
目标单元类型: 调制单元 (相同)
目标参数: 符号率
约束: 调制方式影响符号率范围
```

### 示例2：同链路、跨单元类型
```
链路模式: ACM数传
源单元类型: 编码单元
源参数: ACM使能
目标单元类型: 调制单元 (不同)
目标参数: ACM使能
约束: 编码和调制的ACM状态必须一致
```

### 示例3：通配链路模式
```
链路模式: * (所有链路)
源单元类型: 编码单元
源参数: 编码方式
目标单元类型: 编码单元
目标参数: 编码码率
约束: 通用的编码方式影响码率范围
```

## 术语对比

| 旧术语 | 新术语 | 说明 |
|--------|--------|------|
| 单元名称 | 链路模式 | 更专业，避免与"单元类型"混淆 |
| 基带单元 | 基带单元 | 保持不变，指具体的实例 |
| 单元类型 | 单元类型 | 保持不变，指编码/调制/解调/译码 |

## 界面术语统一

### 约束管理界面
- ✅ 源链路模式 / 目标链路模式
- ✅ 源单元类型 / 目标单元类型
- ✅ 源参数 / 目标参数

### 参数配置界面
- ✅ 链路模式：返向中低速数传
- ✅ 单元类型：调制单元
- ✅ 参数：调制方式

### 基带单元管理界面
- ✅ 单元名称：返向中低速数传-调制单元-1
- ✅ 链路模式：返向中低速数传
- ✅ 单元类型：调制单元

## 数据库字段说明

### sys_baseband_param_constraint 表
```sql
source_unit_name VARCHAR(50) -- 源链路模式
source_unit_type VARCHAR(16) -- 源单元类型
source_param_name VARCHAR(64) -- 源参数名称

target_unit_name VARCHAR(50) -- 目标链路模式
target_unit_type VARCHAR(16) -- 目标单元类型
target_param_name VARCHAR(64) -- 目标参数名称
```

### sys_baseband_unit 表
```sql
unit_name VARCHAR(100) -- 单元名称（实例名称）
unit_type VARCHAR(16) -- 单元类型（ENCODE/MODULATE/DEMODULATE/DECODE）
-- 注：这里的 unit_name 是实例名称，不是链路模式
```

## 总结

- **链路模式**：工作模式，8种选项，决定整体流程
- **单元类型**：功能模块，4种类型，处理链路中的一环
- **参数**：可配置属性，每个单元类型有多个参数
- **约束**：参数之间的依赖关系，可以跨单元类型

这样的术语体系更加清晰、专业，避免了"单元"一词的歧义。
